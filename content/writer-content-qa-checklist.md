# Writer Content QA Checklist

Use this checklist before publishing any new or updated help article.

## Structure

- Includes frontmatter with `title`, `description`, `category`, `categorySlug`, `tags`, `publishedAt`, `updatedAt`.
- Uses the standard sections: `Overview`, `Before you begin`, `Steps`, `Verification`, `Need help?`.
- Keeps numbered steps in logical order with no duplicates.

## Clarity and Tone

- Uses short, actionable instructions.
- Avoids unsupported claims and vendor marketing language.
- Uses consistent terms (`POS`, `Management Page`, `KDS`, `customer-facing display`).
- States model applicability when hardware-specific.

## Screenshot Planning

- Includes screenshot placeholders where visual guidance is needed.
- Placeholder note text clearly explains exactly what to capture.
- No leftover draft bracket notes like `[insert screenshot ...]`.

## Metadata and Category Quality

- `category` matches one of the site categories.
- `categorySlug` maps correctly to the category route.
- Description is concise and readable in cards/search previews.
- Tags are relevant and searchable.

## Verification Quality

- Contains concrete validation checks the reader can perform.
- Includes expected result after critical steps.
- Mentions escalation path when issues persist.

## Final Content Checks

- No broken links or placeholder URLs.
- No grammar/spelling issues in titles or headings.
- Date fields are valid ISO format.
- Article renders correctly in local preview and search results.
